## Project overview

BYUI Spring 2026 dbt capstone project. Stack: dbt Core + Snowflake (data warehouse) + Apache Airflow (orchestration). Key-pair authentication is used throughout — no password/browser auth — because DAGs run inside Docker where no browser is available.

Reference repos:
- Capstone template: https://github.com/atrivedi-dbtlabs/capstone_template
- BYUI Airflow setup: https://github.com/byui-db/Apache_Airflow

## Snowflake connection

Credentials live in `.env` (git-ignored). Connection uses key-pair auth:

- `SNOWFLAKE_PRIVATE_KEY_PATH` points to the local private key file (no `.pub`)
- Account format: `ORG-ACCOUNT` (e.g. `SFEDU02-ZAB82778`)
- Default database: `SNOWBEARAIR_DB`, schema: `RAW`, warehouse: `ANIMAL_TASK_WH`, role: `TRAINING_ROLE`

## dbt commands

```bash
dbt debug          # verify Snowflake connection
dbt run            # run all models
dbt test           # run all tests
dbt run -s <model> # run a single model
dbt test -s <model> # test a single model
dbt build          # run + test together
dbt docs generate && dbt docs serve  # generate and browse docs
```
